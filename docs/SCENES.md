# 场景模式设计

分支：`feature/scenes`

## 目标

在 IM 聊天之外增加一个“同场景共处”的模式。用户和体育生不只是发消息，而是在同一个场景里行动。

## 核心结构

- Scene Agent：负责场景描写、环境变化、快捷行为选项。
- Straight Bro Agents：仍然保持各自人设，但在场景模式下允许动作描写。
- IM Panel：场景模式下仍可独立打开 IM 消息。

## 场景示例

- 宿舍
- 操场
- 篮球场
- 夜宵摊
- 便利店
- 健身房
- 雨天校门口

## 场景 Agent 职责

- 描写当前环境
- 控制节奏
- 给出 3 到 5 个快捷行为选项
- 避免替用户决定
- 让体育生动作符合人设

## 体育生 Agent 在场景模式下新增规则

- 可以写轻量动作描写
- 动作要短，像真实生活
- 不要小说化长段描写
- 不替用户行动
- 不改变“兄弟”关系基调
